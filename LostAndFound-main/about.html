About Campus Lost & Found

Problem Statement

Students often lose their belongings inside the college campus (ID cards, wallets, bottles, etc.). There is no centralized digital system to help them report or recover items easily.

Objectives

  • Provide a simple interface to report lost items with photos.
  • Allow reporting of found items without exposing detailed photos.
  • Enable browsing of lost & found lists in one place.
  • Build a frontend prototype that can later be connected to a backend.

Scope (Frontend Only)

This project currently includes only HTML/CSS/JS, using dummy data and optional localStorage for demonstration. No real login or database.

How the System Works

  • Lost item owner submits a form with photo and details.
  • Finder submits a form without photo.
  • Both can check the lists on the “View Items” page.
  • In real life, they meet and the finder asks the claimant to correctly describe the item before returning it.

Future Scope

  • Add backend with Node.js / PHP and a real database (MySQL).
  • Implement user authentication via college email.
  • Admin panel to approve or delete entries.
  • Email notifications or SMS alerts.
  • AI-based matching between lost and found items.

Technologies Used

  • HTML5, CSS3, JavaScript
  • LocalStorage for demo data
